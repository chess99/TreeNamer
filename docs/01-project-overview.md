# TreeNamer 项目概述

## TreeNamer 简介

TreeNamer 是一个专门用于批量重命名和重组目录结构的桌面应用程序。它提供了一种简单直观的方式来编辑和操作复杂的文件系统层次结构，特别适合需要重组大型项目、资源库或文档集合的用户。

### 目标

1. **简化批量重命名**：提供直观的界面，使复杂的文件和目录重命名操作变得简单。
2. **提高工作效率**：通过文本编辑方式快速修改整个目录结构，避免手动逐个重命名文件。
3. **减少错误风险**：通过预览更改，减少批量文件操作的风险。

### 核心价值

- **效率**：几分钟内完成可能需要数小时的手动文件操作
- **准确性**：减少手动重命名错误
- **可视化**：清晰展示目录结构及变更
- **跨平台**：支持 Windows、macOS 和 Linux

### 关键功能

- **树形结构可视化**：以树形图展示文件和目录结构
- **文本编辑模式**：通过简单的文本编辑来重命名和重组文件
- **更改预览**：应用前查看将要进行的更改
- **智能分析**：自动检测和解决潜在的文件命名冲突
- **系统保护**：防止对系统关键目录进行意外修改

## 项目愿景

TreeNamer 旨在解决以下问题：

1. **简化文件组织**：通过可视化和编辑目录结构，使文件组织更加直观和高效。
2. **批量重命名**：提供强大的工具，使用户能够一次性重命名多个文件和目录。
3. **减少错误风险**：通过预览更改和自动备份，减少批量文件操作的风险。

## 核心功能

TreeNamer 的主要功能包括：

- **目录树解析**：快速解析和显示目录结构
- **双模式编辑**：支持文本和树形视图编辑模式
- **即时预览**：在应用更改前查看所有操作的预览
- **智能重命名检测**：准确识别文件和目录的重命名操作
- **自动备份**：在应用更改前自动创建备份
- **恢复机制**：简单的界面用于浏览和恢复备份
- **跨平台支持**：适用于 Windows、macOS 和 Linux

## 项目结构

项目的主要组件包括：

```
TreeNamer/

└── tests/                # 测试文件
```

## 系统架构

TreeNamer 采用以下架构：

1. **Rust 后端**：处理文件系统操作、目录解析和变更计算
2. **React 前端**：提供用户界面和交互
3. **Tauri 桥接**：连接 Rust 和 JavaScript 层
4. **Zustand 状态管理**：管理应用状态和前后端通信

该架构充分利用 Rust 的性能和安全性处理文件系统操作，同时使用 React 提供现代化的用户界面。
